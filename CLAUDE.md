# Expense Tracker — Claude Context

## Project Overview
A personal expense tracking app built with **Next.js** and **Supabase**.

## Stack
| Layer      | Tech                          |
|------------|-------------------------------|
| Frontend   | Next.js (App Router)          |
| Backend    | Supabase (Postgres + Auth)    |
| Language   | TypeScript                    |
| Styling    | Tailwind CSS                  |

## Database Design

### `categories`
| Column    | Type    | Notes                                      |
|-----------|---------|--------------------------------------------|
| id        | UUID PK | auto-generated                             |
| name      | TEXT    | e.g. "Food"                                |
| icon      | TEXT    | emoji string e.g. "🍔"                    |
| user_id   | UUID?   | NULL = default; set = user custom category |

### `expenses`
| Column      | Type         | Notes                    |
|-------------|--------------|--------------------------|
| id          | UUID PK      | auto-generated           |
| user_id     | UUID FK      | → auth.users             |
| category_id | UUID FK      | → categories             |
| amount      | NUMERIC(12,2)| must be > 0              |
| description | TEXT?        | optional note            |
| date        | DATE         | defaults to today        |
| created_at  | TIMESTAMPTZ  | server default           |
| updated_at  | TIMESTAMPTZ  | auto-updated via trigger |

## RLS Policy Summary
- **categories SELECT**: `user_id IS NULL OR user_id = auth.uid()`
- **categories INSERT/UPDATE/DELETE**: `user_id = auth.uid()` only
- **expenses SELECT/INSERT/UPDATE/DELETE**: `user_id = auth.uid()` only

## Auth Flow
- Auth is handled via **Supabase email + password**.
- Server Actions (`app/auth/actions.ts`) use `@supabase/ssr` — never the browser client.
- After signup, Supabase sends a confirmation email; user lands on `/auth/verify`.
- After login, user is redirected to `/dashboard`.
- `useActionState` drives the loading/error state in `AuthForm` — no extra state management needed.
- Error messages are normalised in the action before reaching the UI.

## Supabase Client Pattern
| Context                      | Import from                  |
|------------------------------|------------------------------|
| Server Components / Actions  | `@/lib/supabase/server`      |
| Client Components            | `@/lib/supabase/client`      |

## Key Conventions
- All IDs are UUIDs generated by Postgres (`gen_random_uuid()`).
- Monetary amounts stored as `NUMERIC(12,2)` — never use `FLOAT` for money.
- Dates stored as `DATE` (no time); timestamps use `TIMESTAMPTZ`.
- TypeScript types live in `src/types/database.ts`; keep them in sync with the schema.
- Default categories have `user_id = NULL` and must not be editable by users.
